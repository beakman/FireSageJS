query rules:
1. Query: When ordering by key, you may only pass one argument to startAt(), endAt(), or equalTo(). // actually include startAfter and endBefore
2. When ordering by key, the argument passed to startAt(), startAfter(), endAt(), endBefore(), or equalTo() must be a string.
3. You can't combine multiple orderBy calls.
4. cursor does nothing without orderBy, ban cursor without orderBy
5. When ordering by priority, the first argument passed to startAt(), startAfter() endAt(), endBefore(), or equalTo() must be a valid priority value (null, a number, or a string).

key naming rule:
Firebase keys must be non-empty strings and can't contain ".", "#", "$", "/", "[", or "]").
